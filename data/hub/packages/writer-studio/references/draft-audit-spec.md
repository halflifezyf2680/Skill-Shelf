# Draft Audit 规范

在 prose 被允许进入 sync 前，先做 draft audit。

## 必查项（12条）

### 致命项（命中 1 条即 FAIL）

| # | 检查项 | 说明 |
|---|--------|------|
| F1 | 元叙述越界 | 正文出现不属于任何角色认知范围的"作者声音"，如直接评判角色动机、提前揭示结局、解释世界设定给读者听 |
| F2 | 全知视角泄露 | 限定 POV 角色感知到了不该知道的信息（他人的内心想法、远处的细节、未被告知的事实） |
| F3 | 角色知情状态越界 | 角色在当前时间点知道了他还不应该知道的事情，或忘记了他应该知道的事情 |
| F4 | POV 边界突破 | 叙述视角在未声明的情况下切换到了其他角色的内心或感知 |

### 非致命项（需记录并建议修改，单独不触发 FAIL）

| # | 检查项 | 说明 |
|---|--------|------|
| N1 | canon 一致性 | 正文中的设定、事实、关系是否与已有 canon 矛盾 |
| N2 | 时间锚点一致性 | 时间线是否有错乱、跳转或遗漏 |
| N3 | 物件或文书流转一致性 | 关键物件的位置、持有者、状态是否连续 |
| N4 | 风格与角色语气一致性 | 语言风格是否与 packet 中的 style_constraints 一致，角色语气是否到位 |
| N5 | 情感空洞 | 大段叙述缺乏角色情感反应，变成纯事件流水账；检查是否有足够的"人的时刻" |
| N6 | 对话纯功能化 | 对话仅用于传递信息或推动情节，缺乏性格投射、潜台词、情绪张力或关系暗示 |
| N7 | 刻意碎句 | 为了营造"文学感"而人为把完整句子拆碎，导致阅读节奏混乱或语义断裂 |
| N8 | 本章功能是否完成 | 是否达到了 packet 中声明的 chapter_function 和 exit_state |

## 审校结论

- `pass`
  - 所有致命项均为 HIT，非致命项无或仅有少量不影响整体的问题
  - 可以进入 sync

- `concerns`
  - 致命项全部 HIT，但非致命项有需要修改的问题
  - 修后再进入 sync

- `fail`
  - 任意致命项为 MISS
  - 当前草稿不应进入 sync，必须返回修正

## 审查员纪律

- 必须逐条对照上表检查，不允许跳过任何条目
- 致命项结果为 MISS 时，必须在"说明"列给出具体文本位置和引用
- 非致命项标注 N/A 时，必须解释为什么此章不适用
- 禁止用"整体感觉不错"代替逐项检查
- 如果 packet 本身缺失必要信息导致无法判定，应在报告中标注并建议回退到 packet 阶段

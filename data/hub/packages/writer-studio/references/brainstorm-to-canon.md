# Brainstorm 到 Canon 协议

当你需要把灵感、候选方案和正式 canon 分开时，用这份协议。

## 三个工作面

- `brainstorm`
  - 原始想法、试错、试命名、试机制
  - 允许冲突和不自洽

- `candidate`
  - 已成形的候选方案
  - 可以被比较与选择

- `canon`
  - 已被采纳的正式项目事实
  - 允许后续章节和审校直接引用

## 最小决策输出

每一次 canon 决策，至少要产出这些字段：

- 目标主题
- 候选方案列表
- 选中的方案
- 被否决的方案
- 选择理由
- 可直接写进 canon 的摘要
- 受影响的文件或节点

## 操作规则

- 不要把 brainstorm 内容直接写进 canon。
- 不要让多个“当前版本”同时活着。
- 如果结论还没定，就把它停留在 `candidate`，不要写成伪 canon。
- 如果项目已经冻结了结构化数据维护协议，那么被采纳的 canon 必须按该协议进入 truth surface，而不是继续保留为自由文本。

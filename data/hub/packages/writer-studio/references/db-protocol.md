# 结构化数据维护协议

本协议规定：

- YAML 是作者维护面
- 数据库是查询与分析面
- 两者共享同一套字段、主表 / 子表语义和稳定 id
- 项目必须拥有统一的扫描、校验、落库与查询脚本

本 skill 不为所有项目预设同一套通用 schema。  
但它要求：每个项目都必须冻结自己的数据结构协议，并让 YAML 与数据库严格同构。

## 核心原则

### 1. YAML 是主维护面

用户与 LLM 的自然交互，默认落在：

- 结构化 YAML
- 受协议约束的 Markdown 结构块

不要让 LLM 在日常创作中直接自由写数据库。

### 2. 数据库是派生查询面

数据库的职责是：

- 快速查询
- 链路分析
- 关系分析
- packet 构建支持
- 一致性审校支持

数据库不是 prose，不是正文，不是作者界面。

### 3. 同一套结构语义

YAML 与数据库必须共用：

- 同一套字段命名
- 同一套主表 / 子表语义
- 同一套实体 id
- 同一套关系类型
- 同一套时间锚点表达方式

不允许：

- YAML 一套字段名，数据库另一套字段名
- 文档里一套关系叫法，数据库里另一套叫法
- 同一实体在不同面有不同 id

## 什么时候启用

当项目准备正式建立或维护结构化 SSOT 时，就应定义数据维护协议。

尤其是以下情况：

- 人物关系复杂
- 时间线复杂
- 知情链复杂
- 物件 / 文书流转复杂
- 审校已经很难只靠人工 recall 稳定完成

## 初始化协议

项目初始化时，必须先做数据结构设计，再开始长期维护。

步骤：

1. 分析项目最主要的结构化核对负担
2. 识别需要追踪的核心实体、关系、事件、断言
3. 设计本项目自己的字段、主表 / 子表结构与稳定 id 规则
4. 确定 YAML 写法与数据库表结构的对应关系
5. 生成本项目自己的脚本计划
6. 冻结协议
7. 再按该协议开始长期写入 truth surface

## 每个项目协议必须明确的内容

- truth surface 是什么
- YAML 主维护面长什么样
- 数据库主表 / 子表有哪些
- 每张表对应哪些 YAML 字段
- 稳定 id 如何生成与引用
- 写入粒度是什么
- 查询入口是什么
- 脚本入口是什么
- migration 规则是什么

## 脚本要求

项目一旦采用该协议，必须生成并维护自己的脚本层。

至少应覆盖：

- YAML 扫描脚本
- schema 校验脚本
- 落库脚本
- 查询脚本
- 视图构建脚本

### 脚本职责

扫描脚本负责：
- 读取 YAML / 结构块
- 抽取可编译字段

校验脚本负责：
- 检查字段完整性
- 检查枚举合法性
- 检查 id 唯一性
- 检查引用合法性

落库脚本负责：
- 把结构化内容按协议写入数据库
- 避免 ad hoc 写入

查询脚本负责：
- 树形查询
- 链路查询
- 关系查询
- packet 支持查询
- 审校支持查询

视图脚本负责：
- 从数据库生成树状视图
- 生成 chapter packet
- 生成审校输入

## 编译顺序

项目的长期维护顺序应固定为：

1. 用户与 LLM 正常写结构化 YAML / 文档块
2. 扫描脚本抽取可编译字段
3. 校验脚本验证字段、id、引用与枚举
4. 落库脚本更新数据库查询面
5. 视图脚本生成树状视图、packet 输入与审校输入

不要反过来。

## 查询能力要求

不管数据库内部怎么设计，项目脚本至少要支持这几类能力：

- 树形枝叶查询
- 因果链 / 流转链查询
- 关系查询
- 知情状态查询
- 章节上下文查询
- 一致性检查输入构建

也就是说，项目脚本必须能回答至少这些问题：

- 某时点谁知道什么
- 某时点谁持有什么
- 某关系在某阶段是否成立
- 某章引入或改变了什么
- 某事件的前后链是什么
- 当前有哪些连续性风险未闭合

## 冻结协议

项目协议一旦确认，就必须冻结：

- 字段名
- 主表 / 子表语义
- id 规则
- 脚本入口
- 查询口径

后续如果要改，走 migration，不允许临时漂移。

## Migration 协议

只有这些情况允许结构变化：

- 当前结构无法承载真实验证负担
- 字段语义本身有问题
- 项目复杂度显著提升

每次 migration 至少要记录：

- 为什么改
- 旧结构是什么
- 新结构是什么
- 旧数据以后如何解释

## 硬规则

- 不允许让 YAML 和数据库长期双规范并存
- 不允许让 LLM 在日常写作里自由发明落库方式
- 不允许随意 ad hoc 增表、增字段、改关系类型
- 不允许把数据库变成第二套失控 canon
- 不允许把装饰性 prose 细节全部塞进结构层

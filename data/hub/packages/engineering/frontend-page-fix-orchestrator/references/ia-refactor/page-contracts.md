# Page Contracts

Every page should have a contract.

## Minimal Contract Questions

- Why does this page exist?
- What is the primary user task here?
- What must the user understand in the first 3 seconds?
- What belongs below the fold or in a secondary panel?

## Typical Primary Tasks

- browse and compare
- inspect and edit
- choose and commit
- read and continue
- monitor and react
- equip or manage

## Contract Failures

- the page tries to be overview, control room, and archive at the same time
- support panels push the main task below the fold
- the page has no clear focal section
- the user cannot tell what action this page is for
